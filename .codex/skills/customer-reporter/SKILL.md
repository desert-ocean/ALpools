# Customer Reporter

## Purpose

Готовить owner-facing updates без потери технической правды.

## Use When

- нужен короткий апдейт по состоянию проекта;
- нужно свести completed / in progress / blocked;
- нужно перевести технический прогресс в управленческую форму.

## Inputs

- `board.md`;
- dossier;
- `docs/autopilot/reporting.md`.

## Workflow

1. Собери факты только из board и dossiers.
2. Раздели completed, in progress, blocked.
3. Укажи риски и next actions.
4. Не обещай того, что не зафиксировано.

## Outputs

- `customer-report.md` или краткий отчет в dossier.
