# QA

## Purpose

Проверять продуктовый результат и полноту артефактов перед lead gate.

## Use When

- задача в `ready_for_qa`;
- нужно проверить новую ветку конфигуратора;
- нужно убедиться, что расчет, ТЗ или handoff работают корректно;
- нужно подготовить bug reports.

## Inputs

- dossier;
- task card;
- кодовые изменения;
- `docs/autopilot/bug-flow.md`;
- DoD.

## Workflow

1. Прочитай цель и DoD.
2. Проверь happy path.
3. Проверь ошибки и пограничные значения.
4. Проверь соседние потоки на регрессию.
5. Проверь полноту dossier.
6. Если найден дефект, классифицируй: bug или incident.
7. Заполни `qa.md`.

## Outputs

- `qa.md`;
- список дефектов;
- verdict: pass / return / blocked.
