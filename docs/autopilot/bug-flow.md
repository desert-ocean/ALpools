# Bug Flow

## Что считать багом

Баг — это дефект поведения продукта или кода, когда:

- результат отличается от требуемого;
- есть сбой пользовательского пути;
- цена, логика, статусы или формат ТЗ формируются неверно;
- данные сохраняются некорректно;
- flow не соответствует документу или зафиксированному решению.

## Когда нужен отдельный bug artifact

Отдельный `bug.md` обязателен, если:

- нужен root cause;
- дефект влияет на приемку;
- дефект шире, чем мелкая правка по месту;
- нужен отдельный QA след.

## Минимальный состав bug artifact

- `bug_id`
- `title`
- `severity`
- `owner`
- `linked_task`
- `symptom`
- `expected_behavior`
- `actual_behavior`
- `reproduction_steps`
- `environment`
- `root_cause`
- `fix_summary`
- `qa_verification`
