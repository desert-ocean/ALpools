## Summary

Добавить изолированную Telegram-админку для управления каталогом реализованных проектов без редактирования кода, с сохранением новых проектов в JSON.

## Problem

Каталог реализованных проектов уже доступен пользователю, но остаётся статическим: для добавления нового проекта нужно менять `app/data/projects.py`. Это не подходит для операционной работы администратора.

## Scope

- Добавить доступ по `/admin` только для `ADMIN_IDS`.
- Создать изолированные файлы `app/handlers/admin_projects.py`, `app/states/admin_states.py`, `app/keyboards/admin_kb.py`.
- Реализовать FSM `AdminAddProject`.
- Сохранять новые проекты в `data/projects.json`.
- Сохранить совместимость пользовательского каталога через существующие функции доступа к проектам.
- Реализовать админский список проектов.
- Оставить `Удалить проект` и `Редактировать` заглушками.

## Out Of Scope

- Полноценное редактирование проекта.
- Реальное удаление проектов.
- Перестройка пользовательских handlers каталога.
- Полная миграция legacy-каталога из Python в JSON.

## Definition Of Done

- Команда `/admin` доступна только пользователю из `ADMIN_IDS`.
- Админ-меню показывает действия: добавить проект, список проектов, удалить проект.
- FSM `AdminAddProject` собирает: название, город, размер, тип, описание, фото, подтверждение.
- На шаге фото принимается несколько изображений и сохраняются `file_id`.
- Кнопка `Готово` без фото не переводит дальше.
- Новые проекты сохраняются в `data/projects.json` с автоинкрементным `id`.
- Пользовательский каталог не ломается и продолжает работать через `get_projects()` / `get_project_by_id()`.
- Админский список показывает название и ID проекта, а также кнопки `Редактировать` / `Удалить` как заглушки.

## Review Requirements
