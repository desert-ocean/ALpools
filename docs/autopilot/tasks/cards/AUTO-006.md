## Summary

Расширить админ-панель каталога проектов: дать возможность редактировать и удалять JSON-проекты, а также публиковать проект в Telegram-канал автоматически после добавления и вручную по кнопке.

## Problem

После `AUTO-005` администратор может только добавлять проекты. Кнопки редактирования и удаления остаются заглушками, а публикация проекта в канал отсутствует. Это оставляет операционный контур незавершённым и требует ручных правок и дублирования действий.

## Scope

- Реализовать edit-flow для JSON-проектов каталога.
- Реализовать delete-flow с подтверждением.
- Реализовать FSM `AdminEditProject`.
- Добавить редактирование полей: название, город, размер, тип, описание, фото.
- Поддержать операции с фото: заменить, очистить, добавить новые.
- Добавить кнопку `📢 Опубликовать заново` в админский список.
- Добавить автопубликацию после успешного добавления проекта.
- Добавить безопасную конфигурацию `CHANNEL_ID` и `try/except` вокруг публикации.
- Сохранить текущую логику пользовательского каталога и существующего add-flow.

## Out Of Scope

- Полная миграция legacy seed-проектов из `app/data/projects.py` в JSON.
- Редактирование legacy seed-проектов через Telegram UI.
- Изменение пользовательских callbacks каталога `project_*`.
- Перестройка архитектуры хранения за пределами файлового JSON-слоя.

## Definition Of Done

- В админском списке у каждого проекта доступны кнопки `Редактировать`, `Удалить`, `📢 Опубликовать заново`.
- Для JSON-проекта доступно редактирование всех целевых полей без ручной правки кода.
- Удаление выполняется только после подтверждения.
- Изменения персистятся в `data/projects.json`.
- После добавления нового проекта выполняется попытка публикации в канал.
- Публикация не падает, если `CHANNEL_ID` не задан или Telegram API возвращает ошибку.
- Для проекта с фото публикация отправляет `media_group`, без фото отправляет текст.
- Legacy seed-проекты не ломаются и получают явный безопасный отказ на edit/delete.

## Review Requirements

- Проверить edit/delete только для JSON-проектов.
- Проверить корректность операций с фото: append, replace, clear.
- Проверить автопубликацию и ручную публикацию с фото и без фото.
- Проверить fallback при пустом `CHANNEL_ID`.
- Проверить отсутствие регрессии в add-flow и пользовательском каталоге.
