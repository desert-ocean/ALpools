# Implementation Report

## Objective

Реализовать новый шаг "Дополнительные уточнения" после выбора аттракционов и включить параметр `lighting` в расчёт.

## Inputs

- `analyst.md`;
- `lead.md`;
- `app/handlers/project_configurator.py`.

## Decisions

- Добавлено состояние `ProjectFSM.choosing_lighting`.
- Добавлена отдельная клавиатура выбора освещения.
- Расчёт вынесен в функцию `calculate_price(state_data)`.
- Новые поля `attractions_count`, `lighting`, `price` добавлены в `design` обратносовместимо.

## Work Performed

- Добавлена константа `LIGHTING_PRICE = 10_000`.
- Обновлён FSM и стартовые данные состояния.
- Изменён переход после аттракционов: вместо прямого расчёта показывается новый вопрос.
- Добавлен handler выбора освещения.
- Обновлён расчёт итоговой суммы и breakdown.

## Self-Analysis

- В файле сохранена текущая структура обработчиков, чтобы не переписывать FSM шире задачи.
- Поле `price` заполняется одинаковым total в payload каждого выбранного раздела, потому что текущая модель хранится по разделам, а отдельной итоговой сущности расчёта нет.

## Risks

- Если downstream-код начнет интерпретировать `design["price"]` как цену конкретного раздела, потребуется уточнение контракта.

## Handoff

Передать в `QA` на проверку веток `Да`, `Нет`, `Пропустить` и синтаксической корректности.
