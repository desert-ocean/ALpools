# Analyst Report

## Objective

Превратить запрос на админ-панель управления каталогом проектов в decision-complete задачу для реализации.

## Inputs

- Пользовательский запрос в текущем треде
- `AGENTS.md`
- `docs/autopilot/sources-of-truth.md`
- `AUTOPILOT-FOUNDATION.md`
- `app/data/projects.py`
- `app/handlers/projects.py`

## Decisions

- Хранилище новых проектов выбрать в `data/projects.json`.
- Пользовательский каталог сохранить совместимым через существующие функции доступа к проектам.
- Админский flow изолировать в отдельных `states`, `keyboards`, `handlers`.
- Команду `/admin` ограничить через `ADMIN_IDS = [8369302291]`.
- Функции редактирования и удаления оставить заглушками в рамках текущего scope.

## Work Performed

- Проверена текущая реализация каталога и источник данных.
- Сформирован scope, out of scope и DoD для задачи.
- Зафиксировано решение использовать JSON, а не генерацию Python-кода из Telegram.

## Self-Analysis

Гипотеза: для минимального риска не требуется немедленная миграция legacy-каталога из Python в JSON, достаточно сохранить новые админские записи в JSON и агрегировать их в слое данных. Это нужно перепроверить на этапе QA как решение по совместимости.

## Risks

- Возможна неоднозначность ID между legacy-проектами и новыми проектами, если не нормализовать сравнение идентификаторов в data-layer.
- Нужна ручная проверка UX загрузки нескольких фото внутри Telegram-клиента.

## Handoff

Передать задачу в реализацию как `Backend Bot Engineer`: создать JSON-хранилище, изолированный admin FSM и подключение router без регрессии пользовательского каталога.
